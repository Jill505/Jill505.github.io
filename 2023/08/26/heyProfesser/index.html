# 上半：Game Start！

第一個接觸的遊戲已經想不起是「真三國無雙 4」還是「音速小子」了
但那一款影響我最深的遊戲還記的清清楚楚：說來好笑，那款遊戲是「賽爾號」，一款網頁遊戲。
那時我是中班，頭一次接觸賽爾號時，我整個人都震撼了，居然可以把戰鬥的過程以數值的方式來表現，真是太酷啦！
「居然能這樣玩」的驚奇讓我不亦樂乎，但老媽規定的 30 分鐘哪裡滿足的了我？
到了幼稚園後，我立馬從廢紙堆中創造了我自己的「賽爾號」並邀請同學們一起來玩，那個「遊戲」有的只是幾張在廢棄 A4 影印紙上畫著簡陋的 UI 和幼兒園水平的美術圖

但就是在那時我第一次知道了：「原來看到玩家的笑容，比玩遊戲還要快樂」
在那之後我下定了決心：「我要做遊戲！」

## 國小的紙上遊戲 ##
在這之後一直到國小高年級，我一直做著紙上遊戲，設備也從廢紙升級到了 10 元筆記本
其實當時是想做電子遊戲的，但既沒學習管道也沒資源，那時我甚至連台電腦都沒有，就這樣先把這個念頭放下了
同時在這六年中，我也試著推出許多其他類型的紙上遊戲過，但在安親班與學校的熱度總是不溫不熱，同時在每次「遊戲改版」後玩家的反應也讓我思考一個問題：「自己想做和玩家想要之間的差距」
曾經有一個名為「機器人大戰 (撞名純屬巧合，玩法並不相同)」的遊戲，那是一個玩家必須想盡辦法碰到機器人背部以關閉它的遊戲，而機器人有各種不同的技能
例如：啟動「保護色」時，玩家必須碰到和自己顏色相近的物體、而「雷射」時，必須離開機器人的視野
隨著版本推陳出新，我加入了道具、素材掉落、物品合成...... 甚至是開箱系統，但奇怪的是：遊戲並沒有隨著元素的堆疊而變得越來越好玩，同學的參與度也是越來越低
但還沒來得及想通答案，我便從國小畢業了，從鄉下到城市讀國中。

## 努力補強基礎的國中 ##
一來到學校立馬就來了兩個震撼彈：

  1. 每個人都十分在意成績
  2. 同學都在玩手機，沒人要玩紙上遊戲了

就在這時一個念頭浮現了：「來做電子遊戲吧！」
在有這個想法後我便開始了實際的執行... 才怪咧

做遊戲是一個有一定門檻的，我知道以我當時的實力不可能一蹴而就，一步一步老老實實的把基礎練習好才是夢想大廈最堅實的地基
為此，我將國中三年分成了三個階段，第一年企劃，第二年美術，第三年程式
聽起來自吹自擂，但這三個分類是我在國一的時候自己盯著螢幕想出來的，當幾年後知道開發職務分工也是如此時真的十分驚訝

在這三年間，我的美術水平從
進步到
程式也從鍵盤都不會打進步到基本的物件導向
企劃方面我則透過對每一款我玩到的遊戲去思考「為什麼這款遊戲會好玩？」以及撰寫文章來練習

在練畫、練 code、練文筆的同時，我也沒忘記身為玩家的本份
空閒的時間有大把都給了 Apex 和 Minecraft 這兩款遊戲
其中，Minecraft 的一個多人伺服器改變了我，並且這個改變值得讓它擁有一段篇幅，而它就是：「EOE 上古紀元」

## 影響一生的貴人們：上古紀元伺服器 ##
「上古紀元」是一個 RPG 伺服器，所謂 RPG 伺服器在 Minecraft 中，就是透過修改遊戲中的參數並搭配插件，以達到和原版遊戲不同遊戲體驗的伺服器，換句話來說，也可以算是一種「遊戲中的遊戲」，而 EOE 在當時可以說是全台範圍內最大的 RPG 服
一開始，我在伺服器中只是作為一個單純的玩家，而隨著一次營運團隊的招募，我進入了伺服器的營運團隊中，而在那，我得以用一個不一樣的視角來看待這個世界
上古紀元的營運成員在當時大多是高中或大學，我當時作為一個國中生其實是謊報年齡進去的，在那我見識到了團隊管理、營運的各種大小事，思維方式與措詞也因交友圈的關係變得更加成熟。
那真的是一段十分開心的時光，但隨著那段時間 Minecraft 本身的人口流失，以這款遊戲作為載體的上古紀元伺服器而逐漸到了頭，雖然在這之後伺服器又被幾位創作者救了回來，但對我來說這就是它的結局了。

在三年的沉澱後，我覺得時機成熟了，會考結束後我立刻開始了實際著手開發的準備。

# 後半：開發實戰編年史

在高中生涯中截至 8/27 為止，大體上我共計研發了七款有完整 Game Loop 的遊戲以及為數眾多始亂終棄的專案
每一個作品背後都有一故事以及我所學到的教訓
這些作品與學到的階段性成就和教訓依時序分別是：
賭夜：第一款作品、時間管理
(取消開發) 捷勝時刻：程式碼要整理、Game Loop 的重要性
返祝：內容取捨的重要、項目管理、溝通很重要、每週會議
六家廢土：第一次短時開發、首次移植手機版、上架嘗試
(開發失敗) 快拳：團隊擴增 (8 人)、檢視好運的理所當然、別忽視管理成本
佑可別睡了：逞強一定沒有好結果、時間管理
閃子彈：找回信心、檢視開發速度
業新聞：平衡速度與品質

接下來，便是開發故事的開始了：

# 梭哈！第一款遊戲：賭夜 Gamble Night！

在會考結束後，我正式開啟了項目的籌備，很快的，第一款遊戲「賭夜 Gamble Night」正式立項
由於是第一款遊戲，我打算從門檻比較低的遊戲類型開始嘗試，那就是：文字冒險遊戲

當初的算盤打的倒是很精：

  1. 開學前完成立項期的所有內容
  2. 以自主學習計畫的名義召集同學做遊戲
  3. 慢慢壯大團隊、一步一步挑戰更難的專案
  4. 超越任天堂、大敗索尼、收購微軟

想像很美好，開學後也確實招募到了兩個同學一起參與開發，一位負責程序、一位負責文案
但隨後就遇到了一個我根本沒想過的大問題：負責程序的不會寫程序，文案一直拖搞 (畢竟我們都還只是高中)
到最後的「自主學習」變成了我先完成專案的代碼、教負責程序的寫程序、再告訴他這些代碼什麼意思，最後他自己再把我的代碼抄一遍來完成他的自主學習；
對文案我應要求把故事原案重寫了兩遍，但當他交稿時，距離自主學習計畫結束卻只剩兩個禮拜而已，每天催稿真的讓我感到身心俱疲；
而與此同時，我在同個時間段內負責了全部的美術資源、進行下款遊戲返祝的立項，並且，同時獨自進行著一款為了園遊會而開發的遊戲
實際一套走下來，感覺自己來比和別人一起開發還累

# 街機！？園遊會專案：捷勝時刻

當時正值賭夜正式進入研發期的前夕，高一的十一月有件大事，那就是：校慶園遊會
因為我們學校的黑板是電子黑板的緣故，我有了個想法：「既然有可觸控的設備，那為什麼不拿來做個街機遊戲呢？」
就這樣在班會的時候提議並通過了，由我一個人進行全部內容的開發
這是一款帶有劇情的闖關類街機遊戲，共有六關，以電子黑板的觸控為主要玩法，每一關都有不同的玩法，例如第二關需要點擊和背景顏色相同的怪物、第四關的怪物會瞬間移動......
開發的過程中，我對 Unity 引擎的熟悉度越來越高，但這個專案卻有一個問題： 作為一個街機遊戲，它其實不適合乘載劇情，每一關的差異性又過大，第一次上手很難直覺的了解要如何攻略
而當時正當我還在煩惱如何解決時，園遊會因疫情的關係取消了，這個專案也隨之流產
雖然它流產了，但製作這個專案的過程中依然讓我留了個心眼，讓我往後在規劃時更注重玩法和玩家引導

# 團隊再組成，直指 Rogue like：返祝 Return The Blessing

在賭夜完成後，我記取了教訓，找了另外三個同學重新開始，分別是兩個美術及一個音樂
而這個團隊的組成，也是我這一生中，最幸運的事情之一

返祝的開發過程除團隊溝通時十分激烈以外，協作的氣氛真的非常的不錯
但明明合作順暢，卻有一個大問題：這個專案的玩法過於龐大，以我們的產能根本負荷不了

返祝這款遊戲是一款「Rogue like」類型的遊戲，但我高估了團隊的開發能力
Rogue like 講究的是在同個框架下，不同 Build 組合出的 buff 隨機性、效果相乘的效果，這個玩法要求的是大量的內容來支撐每輪遊戲的變化性
而以團隊當時的水平，無論是企劃品質、程式編碼速度、美術產能都顯示著，當前的規模尚不足以開發如此大型的專案
因此，當一個學期的自主學習告一段落後，我們毅然決然的開啟了一個計劃：小專案計畫

小專案計畫目的在於透過製作許多款不同類型的遊戲來讓團隊的合作磨合更加的順暢，在磨練好後再回頭挑戰返祝
也可以說，小專案計畫的宗旨就是「為了開發返祝而磨練」

# 街機又來！？園遊會專案：六家廢土

時間不知不覺中又過了一年，園遊會的時間又到了，這個時期的疫情已經沒有去年那麼嚴重，我們的開發實力也成長了不少
「去年沒有做到的，今年要加倍奉還！」
抱持著這樣的信念，六家廢土立項了，在這款遊戲中，殭屍將會從遠方不斷往前走，玩家必須在殭屍到達前將其射死，否則就會受傷
建立在此基礎上，我們還製作上了高達六位的可選角色技能、兩隻 boss、以及結合了學校的電子黑板觸控的「體感裝彈系統」

園遊會結束幾天後，因應玩家的反響十分熱烈，我也在生日的那天註冊了 google 的開發者帳號
用了幾天的時間研究如何移植後，六家廢土 M 就正式在 google play 上上線啦

六家廢土在園遊會的成果可以說是大成功，這讓我的自信心增加了可不只是一點半點，甚至都有點自滿了
這個自滿，讓我在接下來的開發過程中，跌了個大跤......

# 團隊大擴增：快拳 Fast Fist！##

根據先前的小專案計畫，快拳應該被設定為一個小體量的遊戲，但自滿的情緒又讓我把餅越畫越大，同時也過於輕視了程序面和管理面的成本與難度
在共同會議後，我們提出的框架是一個「有網路帳號系統的、體感操作的手機平台連線對戰遊戲」

快拳在初期就埋下了兩個隱患

  1. 目標太大
  2. 團隊人數過多

第一點，目標太大
我忽視了連線系統的開發難度，光是 ping 的問題就已經夠讓人頭疼的了，第一次開發連網系統的熟悉度、配對機制、插件熟悉度、和帳號系統對接等等......
我努力了，付出了巨大的努力；
一天 24 小時幾乎全部都全神貫注在開發上，在家 coding、通勤時看插件的 document、在學校規劃團隊進度，連吃飯睡覺都在想有怎樣的寫法可以優化的更好
全部砍掉重寫、寫信詢問插件 API 相關用法、問老師...... 該做的都做了，但我依然分身乏術，大量的惡性 bug 無論付出多少努力都無法消除
技術上，這是作為程序的失職
時間上，這是作為製作人對專案開發規模與難度的掌握不全

第二點，團隊人數過多
在六家廢土在園遊會上展出後，我不顧原本組內反對的聲音進行了擴增招募，一段時間後，團隊人數由 4 人增加至 8 人，新加入的有一位程式、兩位美術和一位企劃
一開始我抱著「人多肯定做事快」的想法，但當我逐漸發現團隊擴增反而增加了溝通成本而導致空轉時間更長時，我開始有點後悔了，但也來不及了
資源調度和工作分配的難度已經成幾何程度上漲，我又為了讓每個人都感到「參與感」而提升團隊的凝聚力，在分配工作時還執意的要讓每個人都有被分配到一部份的內容
什麼都想要的後果就是：新人既沒有學到東西，團隊舊的成員也沒有得到足夠清晰的指示該產出什麼內容
拖稿問題頻發、功能寫不出來、討論沒有交集，連每週固定的會議時間氣氛都越來越僵

如果只有這些就算了，咬著牙還是做得下去，但這時，一個最錯誤的決定做出了：參加為期一個月的巴哈姆特自製遊戲公會 Game Jam

# 自信參戰自製遊戲公會 Game Jam，專案蠟燭兩頭燒：佑可別睡了 York York Don't Sleep

當我在巴哈母特上看見這個活動時就立馬在每週會議上提出並強行通過了
在當時我的判斷下，快拳的開發時間還很充裕，而且正值寒假，當時我估計花一個禮拜左右便可以完成專案，但當回過神來時，這個專案已經花掉了我們一個月的時間，寒假都過去了
雖說在佑可別睡了 (下簡稱 YYDS) 的創作過程中不是沒有收穫，比如說：學到了如何在程式面進行多人協作、Github Branch 的使用方法、和自製遊戲公會的人交流等等寶貴經驗

但是，但是這時候就來了

# 爬的越高，跌的越重：快拳的失敗

所謂人的精力不是無限的，在緊密的連續兩款遊戲加上接連的各種考試讓團隊或多或少出現了產能與參與度雙雙降低的現象，同時又遇上上屆學測結束，我們正式轉變為現役學測生的時期，每週定期會議的氣壓也越來越低，在這樣的各種因素互相影響下，快拳的專案爆炸似乎成為了必然......

在 YYDS 的專案開發結束後，團隊的重心回到了快拳上，然而此時天時、地利、人和都已盡失
天時：精華的寒假時間結束，開學了，大家需要準備各種考試
地利：學校開始鼓勵衝刺學測，班導每天早上固定「學測很重要」的精神喊話
人和：原本的四人經長時間開發疲憊，加入的四人又使分工變得更雜
而這些問題和快拳這個專案本身的問題與被 YYDS 吃掉的時間精力相結合，使得開發失敗率成指數上升

兩個月的時間內，我獨自承受著連線對戰系統龐大的作業量 (另一位程式在負責帳號與 ui 系統)、統整並把控每個人的開發進度、、數值企劃、優化玩法、程式優化等等內容；
時間有限，而在程序和企劃間，我選擇了將更多的精力放在程式上，但這導致了團隊中的各種矛盾更加加劇，衝突也產生的越來越頻繁
還是開頭那句話，人的精力不是無限的，我一個人無法扛下這麼多事，隨著時間推移我的精神開始逐漸衰竭，白話一點講就是：崩不住了

最後，這種狀態在三月達到頂峰，並在 TSGDF2023 後徹底爆發，同時也對團隊成員講了一些十分傷人的話，到現在還是非常後悔
四月，快拳開發中止，我休息了一段時間、好好沉澱並檢討過程中我犯下的錯誤，那一陣子我變得十分消沉、負面，認為自己什麼也做不好、什麼也做不到
那陣子我把身活的重心拉了開來，放了更多的時間在陪伴朋友、家人身上，而在生活的同時，我也發現許多以前想不到的點子像噴泉一般湧出，在創作上反而更起勁、效率更高、更快樂了
這時我才懂了「藝術源自生活」是什麼意思，也才終於聽了做廣告設計的我媽：「創作和生活要平衡」這句話

在休息的這段時間，我發現遊戲開發改變了我，其實起初，我並沒有這麼喜歡程式，總是認為自己是個企劃，本分並不在這，只要找到人接替就馬上丟掉了
但這幾年下來，起出一碰就想逃避的程式卻在不知不覺中已經和寫作、繪畫一樣變成一種享受了
而我也發現，這個改變不只發生在開發的程式上，在各種生活的大小事上也發生了改變
在以前，當我接觸到一件新事物時，總是本能的先拒絕、排斥
但自從進入開發的世界後，頻繁的踏出舒適圈、挑戰從未挑戰過的項目，改變了我的心態；
讓我面對一件從未挑戰的新事物時，首先想的不是「好難、我辦不到」，而是「我該怎麼把這件事變得更有趣」
在以前覺得無聊的事情：生物、地理、建築、舞蹈...... 一切的一切都變得十分有意思！

在好好休息一個月後，端午連假，我產生了一個念頭：
「好久沒做遊戲了，來點 speedrun 吧」

# 速度挑戰，四天連假小清新：連假四天有點閒 順手做個小專案 Dodge Bullet！

「連假四天有點閒 順手做個小專案 Dodge Bullet！」(以下簡稱 DB) 的開發過程十分順利，不如說 DB 在技術面上一開始就沒有設定任何挑戰，唯一的難點只有時間而已
與其說這是一個挑戰，不如說是練手，順便找回被快拳打掉的信心
就在這樣的背景下 UNITY 啟動！
在開發的過程中，我感到了十分順暢的感覺，過程中幾乎沒什麼 bug，一天不到就把本體做完了，剩下時間幾乎都在優化，Unity 彷彿是我身體的延伸，DB 讓我找回了開發的自信

看著同學在玩 DB 時產生的緊張、快樂，那一刻我彷彿又回到了幼兒園，我真心的覺得：「作為開發者，真的太幸福了」

# 一份平衡質、量、時間的考卷：業新聞 Karma News

如果說 DB 是針對製作速度的小考，那麼 Karma News 就像把每科內容都混在一張試卷的期末考
這個專案是為了一個比賽：「全球華人教育遊戲大賽」而製作的遊戲，其實我原本的想法是把 YYDS 稍微改裝一下就拿去參賽，但鄰近交件日一個月，YYDS 並沒有達到作為一個「遊戲」合格的內容量，以遊戲本身的品質來說，如果要參加比賽的話也是對不起評審、對不起團隊、對不起自己。
因此，Karma News 便立項了。

Karma News 的開發時間並不像返祝、賭夜一樣有充裕的時間規劃，製作體量上不能到 DB、六家廢土那麼小品，品質上要避開快拳、YYDS 踩過的坑，立項的時間也正好是期末，說是期末考真的是再適合不過了

說做到就是會做到，一個月過去了，雖然不能說有多好玩、多厲害，但至少比 YYDS 好了
在這款遊戲中，無論是程式、企劃、美術，都能看見過往開發過每一部作品的影子，DB 的程式架構、快拳的 UI、YYDS 的快速產圖、六家廢土的開發節奏、返祝的敘事結構、賭夜的美術風格
在這些過往的基礎上，Karma News 也做出了自己的突破：精細的數值規劃

在準備學測的同時，一款名為「Atomized City: Fatigue」的遊戲已經悄悄立項，等待著作為南台科大學生的我去完成。

# 「結尾：foreach (int i in myFriends){ print (i+"，感謝！"); }」

在經歷了這麼多後，從廢紙堆開始的夢想堅持到了現在，或許能抬起頭來說句：「我也是個遊戲開發者」了
一路以來採的雷多的數不勝數，卻也得到了許多人的幫助，
腳踏實地的開始、自信爆棚、重大失敗、又再找回信心
我是幸運的，我的爸媽沒有叫我去讀醫科，給我自由成長的空間
我是幸運的，能預見上古紀元的一群人，改變我的思維模式
我是幸運的，有一群夥伴再我身邊支持我，忍受我任性的提出各種稀奇古怪的項目要求
我是幸運的，有家人做遊戲企劃、做音校、做插畫、做設計，讓我不覺得沒有人理解我
我是幸運的，有 TSGDF 這樣的舞台，讓我結交志同道合的夥伴

我不是一個天才，所以我無法一步登天，但我腳踏實地精進自己的能力
我不是天生的領導者，所以我曾經做出了許多錯誤的決策，但我從失敗中汲取教訓
我不是個完人，所以再面對巨大的壓力與失敗時我選擇了暫停開發，但我沒有放棄，筆也一日沒有停過

可能有人會質疑，花了這麼多時間和精力到底是為了什麼？
我會回答：「看到玩家的笑容，比玩遊戲還要快樂」

正在撰寫這篇自傳的，不是一個滿腔熱血的青少年，而是一個意志堅定的夢想家。

最後阿，教授給你給聽好了阿，不收我可是貴校的損失，請不要不識抬舉 (^^)
